# LockSafe Webapp - Project Instructions

## Overview
LockSafe UK - The UK's first anti-fraud locksmith marketplace. Next.js app with MongoDB/Prisma, custom JWT auth, agent orchestration system, and Retell AI voice receptionist.

## Tech Stack
- **Framework**: Next.js 16 (App Router)
- **Database**: MongoDB via Prisma ORM
- **Auth**: Custom JWT-based (not NextAuth) - `src/lib/auth.ts`
- **Styling**: Tailwind CSS with slate/orange color scheme
- **Package Manager**: npm (not yarn - uses package-lock.json)

## Key Patterns
- Admin auth uses custom JWT with cookie-based sessions (`auth_token` cookie)
- Admin sidebar at `src/components/layout/AdminSidebar.tsx` - add new pages to `navItems` array
- DB singleton at `src/lib/db.ts` - always import `{ prisma }` from here
- Dynamic route params must use `Promise<{}>` pattern in Next.js 16
- Config values in `src/lib/config.ts` (SITE_NAME, SITE_URL, phone, email)

## Visual Style
- Primary: Orange (#f97316 / orange-500)
- Background: Slate tones (slate-100 for admin, white for public)
- Cards: White with rounded-xl and shadow-sm
- Admin layout: Sidebar (slate-900) + main content area

## Voice AI Receptionist (Retell AI)
- **API Key**: Stored in `.env` as `RETELL_API_KEY`
- **Webhook**: `/api/retell/webhook` - receives call events
- **Calls API**: `/api/retell/calls` - list/filter, `/api/retell/calls/[id]` - detail/update
- **Analytics**: `/api/retell/analytics` - aggregated metrics
- **Config**: `/api/retell/config` - GET/PUT agent configuration
- **Test**: `/api/retell/test-call` - generate test call data
- **Admin UI**: `/admin/voice-receptionist` with tabs: Overview, Calls, Analytics, Settings
- **DB Models**: VoiceCall, VoiceAgentConfig, CallAnalyticsSnapshot
- **Handler**: `src/lib/retell-handler.ts` - processes webhook events, classifies calls
- **Auth**: `src/lib/retell-auth.ts` - HMAC-SHA256 signature verification

## Agent System
- C-suite agents: CEO, CMO, COO, CTO in `src/agents/`
- Tools registry at `src/agents/tools/`
- Core orchestrator at `src/agents/core/`

## Important Notes
- Do NOT modify `next.config.js` - contains security headers
- Build uses `npm run build` 
- Existing schema has ~40+ models - be careful with schema changes
- MongoDB doesn't support migrations - use `prisma db push`
